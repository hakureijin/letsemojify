# §01 Emoji Variant-Complexity Sankey · Design

Date: 2026-05-21

## Goal

Add a third visualisation to §01, below the new CategoryTreemap: an emoji **variant-complexity Sankey** that shows *how* the catalogue grows so large. The cumulative chart answered "how many?". The treemap answered "of what kind?". This chart answers "by what mechanism?" — splitting every entry in the current catalogue (Emoji 17.0, 3,944 fully-qualified emoji) by the variant mechanism that produced it (base, skin-tone modifier, ZWJ family, ZWJ role, hair-style, direction-flipped, etc.) and flowing each mechanism into the 9 Unicode CLDR groups.

## In scope

- `components/chapter-01/VariantSankey.tsx` — new client component
- `data/chapter-01-variants.json` — new data file (generated by the same build script as the category treemap)
- Extend `scripts/build-emoji-categories.mjs` with a `classifyEmojiVariant(codepoints: string)` function and a second JSON output
- Type extensions in `types/chapter-01.ts`
- i18n keys `ch01.variantSankey.*` (zh + en)
- New `d3-sankey` dependency (+ `@types/d3-sankey`)
- README and project-memory updates
- Tests for the data shape and the classifier

## Out of scope

- Year/version slider — the Sankey is a structural snapshot at the latest release. The treemap above already serves the time story.
- Re-doing the §01 category treemap or cumulative chart
- Sankey on §02 or anywhere else

## Component design

### Position
Inside the §01 `<Section>`, directly below `<CategoryTreemap>`, in the same `max-w-6xl px-6` container. A thin separator above mirrors the treemap/treetop break.

### Layout
Two-column Sankey, 880 × 460 viewBox, rendered as inline SVG.

- **Left column — variant mechanism** (~7 nodes, ordered by descending total count):
  1. `base` — single-codepoint emoji with no modifier
  2. `skin-tone` — base + 1 skin-tone modifier
  3. `multi-skin-tone` — sequence with ≥2 skin-tone modifiers (handshakes, kissing/couple variants)
  4. `zwj-family` — ZWJ sequence with ≥2 person codepoints
  5. `zwj-role` — ZWJ sequence with 1 person + activity/object/profession
  6. `hair-style` — ZWJ sequence with a hair-style modifier (1F9B0–1F9B3)
  7. `direction-flipped` — ZWJ sequence with a direction arrow (27A1 / 2194 / 2195 + FE0F)
  8. `zwj-other` — anything else with ZWJ (rainbow flag, etc.) — fold into one bucket

- **Right column — Unicode CLDR group** (9 nodes, same as the treemap)

- **Flows**: one per (mechanism, group) pair with count > 0. Flow width ∝ count. Color: derived from the mechanism's hue (consistent palette), with low opacity on the flow body and higher opacity on hover.

### Interaction model
Same shape as `CumulativeChart`/`CategoryTreemap`/`OriginMap`/`DayInLife`:
- `useState<string|null>` `activeId` + `pinnedId` keyed by `${mechanism}::${group}` (or `node:${id}` for whole-node selection).
- Hover/focus highlights:
  - **On a flow**: dim all other flows; pop a tooltip showing mechanism + group + count + share-of-mechanism + share-of-group + 4 example emojis.
  - **On a node** (mechanism or group): dim all flows that don't touch that node; pop a tooltip showing total count, share of catalogue, and 4 examples.
- Click / Enter / Space pins; outside-click + Escape unpins.
- Tooltips use the same overlay pattern as other charts.

### Labels
- Left nodes: mechanism label + count (e.g. "Skin-tone modifier · 712").
- Right nodes: group label + count (same as treemap labels).
- When a node is <16 px tall, label moves outside the node box.
- No label on flows (they're too small); tooltip carries the detail.

### Accessibility
- SVG `role="img"` with `aria-label` summarising "Sankey of {total} Emoji 17.0 entries flowing from {n} mechanisms into {k} groups."
- Every node and every flow is `role="button" tabIndex={0}` with descriptive `aria-label` ("Skin-tone modifier — 712 emoji, 18% of the catalogue. Press Enter for details.").
- 44 px min hit area on nodes; flows are made hit-friendly via a transparent overlay path twice the visible thickness.
- Visually-hidden `<details>` block lists the full mechanism × group table for screen readers.
- `prefers-reduced-motion`: link entrance animations are removed; only opacity/state changes remain.

### Animations
On mount, links fade in over 600 ms staggered by 30 ms. On hover/focus, opacity transitions in 150 ms. Reduced-motion mode collapses these to 0 ms.

## Data design

### Source
Same `emoji-test.txt` already cached by `scripts/build-emoji-categories.mjs`. No new network call needed.

### Build script changes
The script grows two responsibilities:
1. Existing — classify by *group* and version → `chapter-01-categories.json` (unchanged).
2. New — classify by *variant mechanism* at the latest version snapshot → `chapter-01-variants.json`.

A new exported pure function `classifyEmojiVariant(codepoints: string): MechanismId` lives in the script and is unit-tested. Rule order (first match wins):

| # | Rule | Bucket |
|---|------|--------|
| 1 | Contains `200D` AND any of `1F9B0..1F9B3` | `hair-style` |
| 2 | Contains `200D` AND any of `27A1`, `2194`, `2195` followed by `FE0F` | `direction-flipped` |
| 3 | Contains `200D` AND ≥2 person codepoints (`1F466`, `1F467`, `1F468`, `1F469`, `1F474`, `1F475`, `1F476`, `1F9D1`, `1F9D3`) | `zwj-family` |
| 4 | Contains `200D` AND exactly 1 person codepoint | `zwj-role` |
| 5 | Contains `200D` (and none of the above match) | `zwj-other` |
| 6 | ≥2 skin-tone codepoints (`1F3FB..1F3FF`) | `multi-skin-tone` |
| 7 | Exactly 1 skin-tone codepoint | `skin-tone` |
| 8 | Else | `base` |

Variation selectors (`FE0F`) are stripped before counting.

### Output shape
```ts
interface Chapter01VariantData {
  snapshot: {
    year: number
    versionLabel: string
    total: number
  }
  mechanisms: VariantMechanism[]   // 8 entries, sorted by count desc
  flows: VariantFlow[]              // one per (mechanism, group) pair with count > 0
  source: Source
}

interface VariantMechanism {
  id: MechanismId
  count: number
  share: number      // count / snapshot.total, in [0, 1]
  examples: string[] // up to 4 sample emojis
}

interface VariantFlow {
  mechanism: MechanismId
  group: CategoryGroupKey
  count: number
  examples: string[] // up to 4 sample emojis from this exact pair
}

type MechanismId =
  | 'base' | 'skin-tone' | 'multi-skin-tone'
  | 'zwj-family' | 'zwj-role' | 'zwj-other'
  | 'hair-style' | 'direction-flipped'
```

Source block reuses the same Unicode CLDR URL already cited by the category file.

## Testing

- Shape: `groupOrder` matches the canonical 9-group list (reused); `mechanisms[]` has 8 entries; `flows[]` sums to `snapshot.total`; every flow's `(mechanism, group)` pair is valid.
- Classifier: representative inputs covering each rule branch — `'1F44B'` → `base`, `'1F44B 1F3FB'` → `skin-tone`, `'1F91D 1F3FB 1F3FC'` → `multi-skin-tone`, `'1F468 200D 1F469 200D 1F466'` → `zwj-family`, `'1F468 200D 1F4BB'` → `zwj-role`, `'1F468 200D 1F9B0'` → `hair-style`, `'1F6B6 200D 27A1 FE0F'` → `direction-flipped`, `'1F3F3 FE0F 200D 1F308'` → `zwj-other`.
- i18n parity test will fail until both zh and en have the new keys — addressed.
- Manual: scrub the dev server, focus a flow, pin it, Tab through nodes, switch locales, force reduced-motion.

## Risks & notes

- **Classifier ambiguity.** Some entries fit multiple buckets (e.g., a couple with skin tones is both `multi-skin-tone` AND `zwj-family`). The rule order prioritises the *most informative* label — `zwj-family` before `multi-skin-tone`. Trade-off documented in the build script header and disclosed in the chart caption ("each entry is counted once under its primary mechanism").
- **Sankey readability at small widths.** The 880-wide viewBox scales down, but flows below 1 px get clipped. Mitigation: bottom-stack rare mechanisms; show their detail in the screen-reader `<details>` block.
- **Reading direction.** Some readers expect Sankey flows to read time-flow (past → future). This one doesn't — it's a static decomposition. Caption says so explicitly.

## Follow-up the user should know about

- **Per-version Sankey** — if the time story becomes interesting, a future PR could let the slider on the treemap above also drive the Sankey, animating flows as new mechanisms get added through history (skin-tone in 8.0, hair in 11.0, direction in 15.1). Out of scope today.
