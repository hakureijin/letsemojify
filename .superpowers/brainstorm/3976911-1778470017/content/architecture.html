方案 A · 单页 Scrollytelling

4 章节作为单页面的 4 个 section，顺序滚动。章节间有过渡动画/锚点导航。数据新闻最经典形态（NYT/Pudding/Reuters Graphics）。

+ 故事连贯、阅读流畅、移动友好
− 单页加载偏重，分享细分话题不便

方案 B · 多页站点 (1 首页 + 4 子页)

每个话题独立 URL，首页给目录卡片入口。单独话题可单独分享/收藏。

+ 单话题可深度展开、SEO 友好、分享细分话题方便
− 故事连贯性弱、构建工作量大

方案 C · 单页 SPA + Tab 切换

4 个 tab 在同一页内切换，URL 不变（或 hash 路由）。每次只显示一个话题的可视化。

+ 单页轻量、切换快、可视化单独占满视口
− 故事性最弱、像数据 dashboard 不像数据新闻

我的推荐：方案 A · 4 个话题在时间上有自然递进关系（历史 → 当代标准 → 当代使用 → 未来），scrollytelling 能把这个递进做成"故事"，最契合"数据新闻式"的定位。方案 B 的多页 navigation 会割裂叙事；方案 C 的 tab 切换偏向工具 dashboard。
